#### pass by value(传值)便是调用copy constructor拷贝构造函数的同义词
* 如果一个加法定义传参是传值，那么中途每个传值都会调用一次拷贝构造函数，最后的结果存储在tmp上，返回时继续调用拷贝构造函数传给加法的结果。

#####  初始化和赋值的区别
* 初始化
	* 必须检查其参数的有效性
	* 必须确保数据data被正确初始化，比如string必须最后加一个‘ \0’  null结束符
* 赋值
	* 赋值的目标对象，确保在新资源被赋值过去之前，旧资源必须先行释放（需要调用析构函数销毁旧值，再调用构造函数拷贝新值）
	* 侦测自己赋值给自己的病态情况 **自己赋值自己前，也许就因为上一个原因把自我释放掉了，然后找不到赋值数据）

#### 1.new和delete请对应配合
* s1=new string；
	* delete s1
* s2 = new string[100]
	* delete [ ] s2
* 因为delete有两个动作，一个是对此内存被析构函数调用，第二个是该内存被释放。但是对于如果delete未加 [ ] ，便假设删除对象是单一对象，无法清除干净。

#### 2.四个转义符
* const_cast
	* 将对象或指针的常数性转型掉
	* **在没有mutable情况下**， 对于const C * const this，想要修改this中数据，需要一个间接版Localthis类型转化为 Localthis=const_cast<C * const> (this)，通过Localthis可以修改
	* 对于明知不会修改，但强制要求参数no-const的 可以用该关键词修改
* dynamic_cast
	* 安全的向下转型动作
* reinterpret_cast
	* 函数指针型别转换
* static_cast
	* 最接近传统C的转型动作
###### 尽可能使用const
#本质 以* 为界，const在左边即指针所指目标物为常量，在右边则指指针本身为常量
* 以const修饰函数返回值可以提高安全性
	* 比如 * 乘法 的计算结果以const修饰就可以避免 (a* b)=c; 这种赋值乘法结果的错误情况
* 修饰成员函数，可以重载
	* 例如 对const数据采用特有的构造函数 返回const结果
* 被const修饰的成员函数不会修改任何类中的值则称为bit-wise 位常量合理，通常编译器会坚持位常量而对一些改变类中数值的情况报错，此时数据可以用 #mutable 修饰（即使在const成员函数也可改变
	* 对于没有mutable关键词的可以考虑const_cast
* const  A operator*(const A & rhs) const;
	* 对于第一个const ，表示返回值const
	* 中间const 表示不会修改rhs
	* 最后const（出现在类的成员函数），表示不会修改类的值

#### 2. 尽量以const和inline 取代 #define
**本质**：尽量以编译器compiler取代预处理器 preprocessor
* 因为没有进入符号表，导致编译出错难以定位
	* 没有被编译器看见，直接在预处理解决了
* 例如#define 一个max函数
	* 可能存在max(++a,b) ,其实a可能加了两次，总之内部不可推测
	* 可以改为inline完美替代

#### 3.以new、delete 代替 malloc、free
**本质** ：new和delete对构造函数和析构函数有交互
* malloc只是指向数组空间，并没有对象被初始化
* free直接释放，没有调用析构函数，也许对象本身也有内存配置

##### 4.编译器背后为空类自动生成函数
* copy constructor 拷贝构造函数
* destructor 析构函数
* assignment 赋值运算符
* * 取址运算符
* 如果未声明构造函数，还会建一个 默认构造函数

##### 5.让父类有虚析构函数
**纯虚函数**：在virtual函数声明后加 `= 0` 来定义；必须在派生类中被重写
* 经由 父类指针 删除 一个子类对象， 如果 父类是非虚的析构函数，那么结果未定义！
	* BasePoint * pb= new DerivedPoint（）；
	* 当 delete pb时会报错，通常可能是执行父类的析构函数 导致出问题
	* 解决方案：正是在父类的析构函数设为virtual
* 纯虚  析构函数
	* 因为含纯虚函数的类是抽象类无法被实例化，所以让希望成为抽象类的class申明一个纯虚析构函数
		* 注意： 需要为纯虚 析构函数 提供一个定义式，不然连接器会报错少个符号

##### 6. 友元frien函数警惕使用、menber和non-menber区别
* 区别
	* friend函数可以访问该类的私有成员和保护成员
	* 如果函数必须动态绑定，那么必须是虚函数，虚函数必须是某个类的成员函数
	* 成员函数必须右端引用
* 如果当一个函数f和类class C有关系，可以变为成员menber函数，但以下情况除外
	* operator << \ >> 如果设为成员函数，只能右端引用，比如 s>>cin;
	* 当需要左端引用，且对左端引用需要使用类型转化，也不要menber
